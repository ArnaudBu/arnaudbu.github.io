Coronavirus: when will the pandemic end?

2020-03-25

With the recent COVID-19 pandemic, many countries have imposed very severe lockdowns on their citizens in order to stop the spreading of this highly contagious virus. After several weeks of isolation, everyone has the same question on their mind: 'When will it end?'. Let's try to answer this with some statistics and a little imagination!

As it may seem that we will still be locked down for months, the example of Asian countries and a scrupulous respect of the social distancing rules may end the pandemic sooner that what we might expect. Of course, not all countries will pass through this difficult moment in the same manner, but we may deduce some patterns from those situations that could help anticipate the future of the pandemic.

Data

Sources

For this study, two sources of data will be used:

  • Development of coronavirus from DataHub.io;
  • List of lockdowns on Wikipedia: National responses to the Covid-19 pandemic (csv file with the formatted data available here).

Initial processing and scope

Since we will mainly focus on the development and ending of the pandemic, all data concerning deaths and recoveries will be discarded from the dataset.

Also, we will focus on regions that have had at least 100 contaminations so far (25 March 2020).

Parametric approach

Very few regions have already passed through the crisis, which means that there is a very high uncertainty on areas that are still undergoing it. For this reason, we will try to begin with a parametric approach in order to try to deduce a patter from advanced regions, and then adjust it to regions at the beginning of the virus’ spreading.

Hubei region

The most advanced region is the Chinese province of Hubei, whose capital Wuhan was the starting point of the Coronavirus’ spreading.

Let us first have a look at the number of detected contaminations over time:

The pattern over time seems to be similar to a logistic curve, whose representation is given by the function:

\[ f(x) = \frac{L}{1 + e^{-k(x-x_0)}} \] with:

  • \(L\) the maximum value of the function;
  • \(x_0\) the inflexion time of the curve;
  • \(k\) the coefficient of stiffness of the curve. At the inflexion point, we have a maximum increase of \(\frac{L\times k}{2}\) new cases by day.

We can try to fit this function to the data we have using a simple optimization pattern, such as a Newton’s algorithm.

# Evolution function to calibrate

evol <- function(x, a, x0 , L){
  return(L / (1 + exp(- (x - x0)/ a)))
}

# Add the number of days since beginning of measures

cases[, x:= 1:.N, by = c("Country/Region", "Province/State")]

# Select the Hubei region

hubei = cases[`Province/State` == "Hubei"]

# Compute the parameters

param = optim(c(a = 1, x0 = max(hubei$x), L = max(hubei$Confirmed))
              , function(param){
                #if(any(param < 0)){ return(Inf)}
                sum(abs(evol(hubei$x, param[1],param[2], param[3]) - hubei$Confirmed)^2)
                },
              method="BFGS"
              )
param <- param$par

Of course, this does not mean that that this approach will work for all areas, for the following reasons:

  • Newton’s method needs a convex function in order to converge. The function that we are trying to optimize (sum of the squared differences between the logistic function and the data) is not convex, which means that the algorithm does not necessarily converge to its global minimum. In order to be as close as possible, we set the starting point as close as possible to the expected values for the logistic function parameters;
  • not all areas will follow this pattern: if taken soon enough, social distancing measures can help prevent the exponential pattern at the bottom of the curve. Other measures can also have a huge impact on the curve and are note represented by the data we use: testing campaigns, lockdowns, healthcare system reactivity…

With this approach, we can estimate the end of the pandemic: considering the incubation period to be over after 14 days, we can simply add this period to the date when the threshold of 99.99% of the expected total of contaminated people will have been reached. The 99.99% number doesn’t have any meaning, except being very close of 100%.

\[ day_{end} = x_0 - \frac{1}{k}ln(\frac{1}{99.99\%} - 1) + 14 \]

Applying this formula to Hubei, we get an estimated end date for the pandemic: 2020-04-03.

Extension to other places

Considering this model relevant to other places, we can apply it to get those results:

# Compute the different metrics

calib <- function(x, confirmed, dates){
  param = optim(c(a = 1, x0 = max(x), L = max(confirmed))
              , function(param){
                #if(any(param < 0)){ return(Inf)}
                sum(abs(evol(x, param[1],param[2], param[3]) - confirmed)^2)
                },
              method="BFGS"
              )
  param <- param$par
  return(list(L = round(param[3]),
              x0 = min(dates) + param[2],
              k = round(1/param[1],3),
              date_end = min(dates) + param[2] - param[1] * log(1/0.9999 -1) + 14,
              error_fit = round(mean(abs(evol(x, param[1],param[2], param[3]) - confirmed))/param[3], 3)
              )
         )
}

Non parametric approach

The logistic curve approach we used assumes a very strict pattern for the development of the virus. We can thus wonder if it wouldn’t be possible to deduce the development pattern from the data itself, which would allow to include the outliers in the analysis.

In order to do so, we will fit a simple random forest model to the data. Nevertheless, we will need to make a few assumptions.

  • We will not predict the cumulative number of confirmed cases, but rather the transition ratio \(\frac{x_t}{x_{t-1}}\) with \(x_t\) the number of confirmed cases at time t.
  • Since we haven’t enough data on the recovery side of the curve (diminution of the number of cases), we will try to fit two models: one for the convex part of the curve and one for the concave part. We thus try to define the inflexion point of the curve in order to fit the models. We define it by the first day when at least 7 of the 10 last days have seen a decrease in their daily number of cases.
  • We chose a lag of 14 days for the number of historical values used to predict the next value.
  • We included the number of days since lockdown, when declared in the area, in the model. The list of lockdowns considered, as of the 25th of March is available here (copied from Wikipedia: National responses to the Covid-19 pandemic).

Also, our aim with this model is not to create a predictive model with the best possible performance, but rather to extrapolate the future development in an area from the situations of other areas; thus we will not comply with the traditional machine learning pipeline.

# Import stats on lockdowns
lockdowns <- fread("../../static/img/covid/lockdown.csv")
lockdowns[, ":="(start_date = as.Date(start_date, "%d/%m/%Y"), end_date = as.Date(end_date, "%d/%m/%Y"))]

# Merge with cases data
cases <- lockdowns[cases, on = c("Country/Region", "Province/State")]

# Compute number of days since lockdown

cases[, days_since_lockdown := Date - start_date]
cases[is.na(days_since_lockdown) | days_since_lockdown < 0 | Date > end_date,
      days_since_lockdown := 0]
cases[, c("start_date", "end_date") := NULL]

# Function to compute cumulative differences

cumdiff <- function(x){
  res <- c(NA, x[2: length(x)] / x[1: length(x) - 1])
  res[is.na(res) | is.infinite(res)] <- 0
  return(res)
}

# Compute cumulative differences
cases[order(Date), newconf := cumdiff(Confirmed), by = c("Country/Region", "Province/State")]

# Add lags and build datasets
cases <- cases[order(Date), paste("newconf", 1:14, sep="") := lapply(1:14, function(i) shift(newconf, i)), by = c("Country/Region", "Province/State")]

# Compute two modes: increase phase or decrease phase
cases[, increase := newconf >= newconf1]
cases[is.na(increase), increase := TRUE]
cases[order(Date), decrease_mode := rollsumr(increase, k = 10, fill = 10) < 3, by = c("Country/Region", "Province/State")]
cases[order(Date), decrease_mode := cumsum(decrease_mode) > 0, by = c("Country/Region", "Province/State")]

# Select for training
dataset <- na.omit(cases)
dataset[, tokeep := rowSums(.SD) >= 0, .SDcols = paste("newconf", 1:14, sep="")]
dataset <- dataset[(tokeep)]
dataset[, tokeep := NULL]

# Calibrate the model

library(ranger)

cases.rf.increase <- ranger(as.formula(paste("newconf",paste(c(paste("newconf", 1:14, sep=""), "days_since_lockdown"), collapse="+"), sep = "~")), data=dataset, importance="impurity", keep.inbag = T)

cases.rf.decrease <- ranger(as.formula(paste("newconf",paste(c(paste("newconf", 1:14, sep=""), "days_since_lockdown"), collapse="+"), sep = "~")), data=dataset[(decrease_mode)], importance="impurity", keep.inbag = T)

# Test the fit
pred.rf.increase = predict(cases.rf.increase, cases[complete.cases(cases)], type = "se", se.method = "jack")
pred.rf.decrease = predict(cases.rf.decrease, cases[complete.cases(cases)][(decrease_mode)], type = "se", se.method = "jack")
cases[complete.cases(cases) & decrease_mode, c("mean_pred") := list(pred.rf.decrease$predictions)]
cases[complete.cases(cases) & !decrease_mode, c("mean_pred") := list(pred.rf.increase$predictions)]
cases[!complete.cases(cases) | Confirmed == 0, c("mean_pred", "var_pred"):= list(pmax(newconf,1) , 0)]
cases[order(Date), ":="(mean_cum = cumprod(mean_pred) *min(Confirmed[Confirmed != 0])), by = c("Country/Region", "Province/State")]

This approach, despite its strong assumptions, still has some major shortcomings:

  • since we use a time series model for our predictions, we need to use predictions as variables for other predictions, which increase the uncertainty with time;
  • the uncertainty of predictions is not measured here. Even if we can measure the standard error on a prediction through the model, since our final prediction is the cumulative product of our predictions, that are correlated and of unknow law, it is not possible to have a consistent evaluation for this;
  • the use of a pivot time for model switching makes necessary to include this date as a parameter for the model projection, but we do not know it, which means we will need to test over the possible dates.

In order to run the projection models, we will test different possibilities for the pivot time. Considering the fact that all countries have implemented severe measures in order to fight COVID-19, we will simply assume that the curve will show an inflexion in the next 14 days for all countries. We will thus test all the possibilities for the next 14 days.

# Extend the dataset

CJ.dt = function(X,Y) {
  stopifnot(is.data.table(X),is.data.table(Y))
  k = NULL
  X = X[, c(k=1, .SD)]
  setkey(X, k)
  Y = Y[, c(k=1, .SD)]
  setkey(Y, NULL)
  X[Y, allow.cartesian=TRUE][, k := NULL][]
}

extend_cases <- CJ.dt(unique(cases[, c("Country/Region", "Province/State")]), data.table(Date = as.Date((max(cases$Date) + 1):(max(cases$Date) + 90), origin = "1970-01-01")))
new_cases <- cases[Date == max(Date)]
new_cases <- rbind(new_cases, extend_cases, fill = TRUE)

# Merge with lockdowns
new_cases <- lockdowns[new_cases, on = c("Country/Region", "Province/State")]

# Compute number of days since lockdown

new_cases[, days_since_lockdown := Date - start_date]
new_cases[is.na(days_since_lockdown) | days_since_lockdown < 0 | Date > end_date,
      days_since_lockdown := 0]
new_cases[, c("start_date", "end_date", "x", "mean_cum") := NULL]
new_cases[, ":="(mean_pred = 0, var_pred = 0)]

all_cases <- data.table()

for(b in min(new_cases$Date):(min(new_cases$Date) + 14)){
# Apply model
for(d in unique(new_cases$Date)[-1]){
  new_cases[Date == d, (7:20):= new_cases[Date == d-1, 6:19]]
  if(d < b){
  pred.rf = predict(cases.rf.increase, new_cases[Date == d], type = "se", se.method = "jack")
  } else {
    pred.rf = predict(cases.rf.decrease, new_cases[Date == d], type = "se", se.method = "jack")
  }
  new_cases[Date == d, c("newconf", "mean_pred") := list(pred.rf$predictions, pred.rf$predictions)]
}
  new_cases[, date_turn := b]
  all_cases = rbind(all_cases, new_cases, fill = T)
}

# Make predictions
all_cases[order(Date), ":="(mean_cum = c(1, cumprod(mean_pred[-1])) * first(Confirmed)), by = c("Country/Region", "Province/State", "date_turn")]
all_cases <- all_cases[!(Date == min(Date))]

# Aggregate for testing the sensitivity to the pivot time

sens_pivot <- all_cases[, .(L = max(mean_cum)), by = c("Country/Region", "Province/State", "date_turn")]
sens_pivot <- sens_pivot[, .(L = L / last(L), pivot = date_turn - min(date_turn)), by = c("Country/Region", "Province/State")]
