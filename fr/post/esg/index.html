Implémentation d'un générateur de scénarios économiques

2018-05-15

La librairie esg2: un gse basé sur un modèle G2++

La librairie

esg2 est une libraire sous R implémentant un générateur de scénarios économiques en risque neutre basé sur un modèle de taux G2++. Elle permet de générer des trajectoires de taux stochastiques, ainsi que de projeter un actif via la formule de Black & Scholes.

La librairie peut être installée directement via son repo github:

devtools::install_github(“arnaudbu/esg2”)

Le modèle

Le modèle G2++ a d’abord été proposé dans l’ouvrage Interest Rate Models - Theory and Practice par Mercurio & Brigo. L’idée du modèle est de projeter le taux court instantané comme la somme de deux processus gaussiens corrélés, à laquelle vient s’ajouter une fonction déterministe choisie pour retrouver en moyenne la structure par terme des taux fournie en entrée.

L’utilisation d’une distribution gaussienne permet de simuler des taux négatifs, ce qui était considéré comme un inconvénient à l’époque mais paraît un avantage aujourd’hui.

Les équations

Les équations de projection du taux court r(t) sont les suivantes:

\[ \left\{\begin{matrix} dx(t) = -ax(t) + \sigma dW_1(t) \\ dy(t) = -by(t) + \eta dW_2(t) \\ \rho dt = dW_1(t)dW_2(t) \\ \\ r(t) = x(t) + y(t) + \phi(t) \end{matrix}\right. \]

avec x(0) = y(0) = 0

Le modèle de Black & Scholes implémenté se traduit par l’équation suivante:

\[ \dfrac{dS_t}{S_t} = (r(t) - div) \times dt + \sigma_{BS}dW_S(t) \]

avec div le taux annuel de dividende (ou de loyer dans le cas d’un actif immobilier)

Les courbes des taux

Les courbes des taux sont gérés comme des objets, créés grâce à la fonction:

curvezc()

Les graphes ci-dessus montrent la courbe des taux zéro-coupons de l’EIOPA au 31 décembre 2016 et une courbe des taux swap, qui sert à calculer les prix de swaption.

Les swaptions

Les swaptions sont gérées via la fonction:

swaptions()

à laquelle il faut fournir une courbe des taux swap et une matrice de volatilité, comme illustré ci contre avec des volatilités de Black pour des swaptions de maturités et ténors compris entre 1 et 10 ans.

Le modèle est créé sous la forme d’un objet via la fonction:

g2()

qui prend en entrée les paramètres et la courbe des taux pour la future projection.

Les 5 paramètres du modèle sont calibrés sur la courbe zéro-coupon et les prix de swaptions grâce à la fonction:

calibrate()

qui implémente un algorithme de Nelder-Mead pour l’optimisation. La figure ci-contre montre les résultats du calibrage, avec en trait plein les véritables tarifs et en pointillés les valeurs prédites.

Dans cet exemple, on arrive à une erreur globale d’environ 20%, un des gros défaut du modèle G2++ étant que son problème d’optimisation possède de très nombreux minima locaux.

La projection du modèle de taux se fait avec la fonction:

project()

Après quoi la fonction:

zctable()

permet de récupérer la projection de la courbe des taux à toute date.

Il est alors possible de projeter un actif suivant le modèle de Black & Scholes via la fonction:

bs()

L’exemple ci-dessus présente la projection du taux court, avec la moyenne en trait plein, une trajectoire aléatoire en pointillé, et les quantiles à 1%, 5% et 10% de chaque côté de la distribution. La projection d’un indice action est aussi représentée, à l’aide d’une echelle logarithmique. On voit là un des défauts du modèle de Black & Scholes, qui prend des valeurs potentiellement très éloignées sur un horizon de projection important.

La validation: tests de déflateur et de martingalité

r <- g2model@r
d <- deflator(g2model)
t <- action@s * d
t_mean <- c(action@s0, colMeans(t)*exp(action@div* 1:action@g2model@horizon))
t_sd <- 1.96 * c(0, apply(t,2,sd)*exp(action@div* 1:action@g2model@horizon)) / sqrt(action@g2model@nsimul)

d <- ggplot(data.table(mean = t_mean, s0 = action@s0, conf = t_sd, an = 0:50), aes(x = an)) +
  geom_ribbon(aes(ymin = s0 - conf, ymax = s0 + conf), fill = viol2) +
  geom_line(aes(y = mean, linetype = "Moyenne"), color = jaune) +
  geom_line(aes(y = s0, linetype = "Indice initial"), color = jaune) +
  scale_y_continuous(name = "Indice") +
  scale_x_continuous(name = "", labels = ff(suffix = " ans")) +
  scale_linetype_manual(name = "", values = c("Moyenne" = 1,
                                                         "Indice initial" = 2)) +
  theme_minimal() +
  theme(legend.position = "none")

def_table <- colMeans(deflator(g2model))
r_mean <- zcp2rate(1:length(def_table), def_table, method = g2model@curve@method)
def_table_sup <- g2model@curve@zcp[1:50] + 1.96 * apply(deflator(g2model), 2, sd) / sqrt(g2model@nsimul)
def_table_inf <- g2model@curve@zcp[1:50] - 1.96 * apply(deflator(g2model), 2, sd) / sqrt(g2model@nsimul)
r_sup <- zcp2rate(1:length(def_table_inf), def_table_inf, method = g2model@curve@method)
r_inf <- zcp2rate(1:length(def_table_sup), def_table_sup, method = g2model@curve@method)

p <- ggplot(data.table(mean = r_mean*100, zc_rate = g2model@curve@rates[1:50]*100, conf_inf = r_inf*100, conf_sup = r_sup*100, an = 1:50), aes(x = an)) +
  geom_ribbon(aes(ymin = conf_inf, ymax = conf_sup), fill = viol2) +
  geom_line(aes(y = mean, linetype = "Moyenne"), color = jaune) +
  geom_line(aes(y = zc_rate, linetype = "Courbe des taux"), color = jaune) +
  scale_y_continuous(name = "Taux", labels = ff(suffix = " %")) +
  scale_x_continuous(name = "", labels = ff(suffix = " ans")) +
  scale_linetype_manual(name = "", values = c("Moyenne" = 1,
                                                         "Courbe des taux" = 2)) +
  theme_minimal() +
  theme(legend.position = "none")

subplot(ggplotly(p), 
        ggplotly(d), nrows = 2, shareX = TRUE, shareY = FALSE) %>% layout(annotations = list(
 list(x = 0 , y = 1.05, text = "Test du déflateur", showarrow = F, xref='paper', yref='paper'),
  list(x = 0 , y = 0, text = "Test de martingalité", showarrow = F, xref='paper', yref='paper'))
        )

La projection doit être validée à l’aide de plusieurs tests de cohérence. Le premier étant le test de déflateur, qui consiste à vérifier que la courbe des taux est bien retrouvée en moyenne. La fonction suivant est faite pour:

test_deflator()

On représente en trait pointillé la courbe des taux initiales, en plein la courbe moyenne projetée, et en violet l’intervalle de confiance à 95%.

Le second test, dit de martingalité, consiste à vérifier que la moyenne de la projection du processus actualisé de Black & Scholes est constante. Ce à quoi est dédiée la fonction:

test_martingal()

Le dernier test important, non implémenté dans la librairie, est le test de cohérence avec le marché, qui consiste à retrouver les prix des swaptions à l’aide des trajectoires de taux.
